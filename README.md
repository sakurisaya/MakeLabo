# MakeLabo

メイク好きのための記録・管理Webアプリ。
「なんとなく」になりがちな毎日のメイクを詳細にレシピ化し、再現性と色彩感覚を高めるためのプラットフォームです。

🔗 **[アプリを見る](https://main.d2jqhenkd2g20n.amplifyapp.com/)**  
※ ご確認いただきやすいよう、**ユーザー登録・ログイン不要ですぐに全機能をお試しいただける**ゲストモード（共通テストアカウントへの自動接続）として公開しています。

##  主な機能

- **📍 ピン留めでメイクをレシピ化**
  顔のイラストや写真に対して使用アイテムをピン留めし、「どこに・どう塗ったか」を工程ごとに記録。
- **🎨 PCCS色彩理論による色自動判定**
  コスメの色（カラーコード）を独自のアルゴリズムで解析し、トーン・色相で自動分類。「青みピンク」「ディープトーン」など、直感的な色の言葉で理解を深めます。
- **📊 カラーチップによる視覚化**
  その日使った色をチップとして一覧表示。「なんかまとまりがあったな」「今日はコントラストが強かったな」という印象を、感覚ではなくデータ（色）として確認できます。
- **👗 フリーピン（全身トータルコーデ記録）**
  服やアクセサリーなど、コスメ以外のアイテムもキャンバスに記録可能。トータルコーディネートとしてのメイクを保存できます。
- **🛍 所有コスメの傾向分析**
  カテゴリ別に整理できるのはもちろん、所有コスメの色の傾向をカラーチップ表示で一目で把握。「似た色ばかり買ってしまう」を防ぎ、次の購入の根拠を生み出します。

##  技術スタック

### Frontend
- **React (Vite) / TypeScript**: SPAとして高速な画面遷移を実現し、型安全な開発を徹底。
- **TailwindCSS**: CSSによるスタイリング。モバイルファーストで設計し、モダンなグラスモーフィズムを取り入れたUIを構築。
- **React Router**: クライアントサイドルーティングによるスムーズな画面遷移。

### Backend & API
- **FastAPI (Python)**: パフォーマンスに優れた非同期フレームワークを採用し、RESTful APIを高速に提供。
- **SQLAlchemy / SQLite**: ORMモデルを利用したセキュアなデータ永続化（※ 将来的なスケーリングを考慮し、MySQL等への移行も容易な設計）。
- **Pydantic**: 厳格な型定義によるリクエスト・レスポンスのバリデーション。
- **JWT認証**: パスワードのハッシュ化（Bcrypt）とトークンベースの安全なユーザー認証。

### インフラストラクチャ（AWS）
- **AWS Amplify**: フロントエンドのホスティングと、GitHubと連携した自動ビルド＆デプロイ（CI/CD）環境。
- **Elastic Beanstalk**: バックエンドAPIをNginx + Uvicorn構成のEC2環境上で安定稼働。
- **CloudFront**: AmplifyでホスティングされたHTTPSのフロントエンドから、HTTPのバックエンドへの通信で発生する「Mixed Contentエラー」を回避するためのセキュアなリバースプロキシとして採用。

##  技術的なこだわり・アピールポイント

### 1. PCCS色彩理論の自動判定アルゴリズムの実装
入力されたカラーコード（HEX）をもとに、RGB変換空間から「色相（Hue）」と「トーン（Tone）」を算出し、PCCS（日本色研配色体系）に基づく名称を自動でマッピングする処理をバックエンドに独自実装しました。単なる色の記録にとどまらず、ユーザー自身が色を「言語化」して理論的にメイクを理解できる体験を目指しています。

### 2. ポートフォリオとしての「見せ方」を追求したデスクトップUI
本アプリはスマートフォン（ネイティブアプリ）の手ざわりを意識したモバイルファースト設計ですが、PCからの閲覧（企業の採用担当者様の閲覧）を想定し、**PC表示時には「左側にスマホ画面のモックアップ＋右側にアプリのコンセプト説明」を表示する2カラムの特殊なレスポンシブレイアウト**を独自実装しました。同一のコードベースでプレゼンテーションとアプリ機能の両立を実現しています。

### 3. Mixed Contentエラーを回避するAWSクラウドアーキテクチャ
フロントエンド（HTTPS）と独立したバックエンド（HTTP）をシームレスに通信させるため、AWS CloudFrontをAPIの前に配置しSSL化を構成。クロスオリジン（CORS）問題やセキュリティ制約に躓くことなく、本番環境の構築と切り分けを行いました。

### 4. SPAのUXを損なわない状態管理とスクロール制御
React Routerの遷移時にありがちな「スクロール位置が残ってしまう問題」を解決するため、全体のウィンドウではなくアプリケーション内部のメインコンテナに対して意図的にスクロールリセットフックを実装し、ネイティブアプリに劣らないシームレスで自然なUXを実現しています。
